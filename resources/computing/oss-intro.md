## What Is Open Source Software?

Open Source Software (OSS) refers to software whose source code is made publicly available for anyone to view, modify, and distribute. Unlike proprietary software — where the source code is hidden and controlled by a single vendor — open source software is developed collaboratively and transparently by communities of contributors worldwide.

The term "open source" was formally coined in 1998 by Christine Peterson and adopted by the Open Source Initiative (OSI) to describe a specific set of freedoms associated with software distribution.

---

## The Four Essential Freedoms

The Free Software Foundation defines the following essential freedoms that open source software should grant to its users:

| Freedom | Description |
|---------|-------------|
| **Freedom 0** | The freedom to run the program for any purpose |
| **Freedom 1** | The freedom to study how the program works and modify it |
| **Freedom 2** | The freedom to redistribute copies to help others |
| **Freedom 3** | The freedom to distribute modified versions of the program |

## Why Open Source Matters

### For Education
- Students can inspect and learn from real-world code
- Teachers can customize educational tools without licensing fees
- Institutions avoid vendor lock-in and high software costs

### For Government
- Source code can be audited for security and backdoors
- Reduces dependence on foreign proprietary technology
- Supports transparency and public accountability

### For Developers
- Massive ecosystem of reusable libraries and tools
- Meritocratic contribution model rewards quality work
- Global collaboration across institutions and borders

## Getting Started with OSS

1. **Find a project** — Browse [GitHub](https://github.com/explore) or [GitLab](https://gitlab.com/explore)
2. **Read the README** — Understand what the project does and its goals
3. **Check the license** — Make sure you understand your rights and obligations
4. **Find an issue** — Look for issues labeled `good first issue` or `help wanted`
5. **Read CONTRIBUTING.md** — Follow the project's contribution guidelines
6. **Submit a pull request** — Propose your changes for review
